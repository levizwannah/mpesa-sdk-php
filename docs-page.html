Introduction Last updated: 2022-08-18

This SDK provides an abstraction over the Safaricom Mpesa API in PHP. Most of the routine tasks required when making a requests to the API is handled by the SDK. Using the SDK is mostly: You provide data, the SDK makes the request, and you handle the response.

This Documentation teaches you how to use the various APIs provided by the SDK. It also covers the naming conventions, the code, and how to contribute.

Whetting your appetite

The SDK

This SDK provides an elegant coding style, making it intuitive, easy to understand and use. It was developed to operate independently of other libraries. Therefore, it can be used in a standalone way without bringing other libraries along.

Due to lack of dependency, you can use this SDK in any PHP project: framework based or plain old vanilla PHP. This also makes it compatible with PHP 7 and above.

Get it: composer require levizwannah/mpesa-sdk-php

Installation

Requires PHP 7.4 and above

Use composer to import this SDK directly into your project. However, if you are not using composer, download the code from github and use the self-autoload.php file to auto load the SDK files.

Using Composer

If you are using composer, use the command below to import the SDK into your project.

composer require levizwannah/mpesa-sdk-php

Note

This is the recommended approach to import the SDK into your project. If you do not have Composer installed, get it here.

Direct Download

If you are not using Composer, visit Github and download the zipped version of the SDK here. After that, extract the downloaded zip. In the extracted folder's root, you will see self-autoload.php. Ensure that you include this file whenever you are using the SDK. 

    <?php
      require(__DIR__ . "/path-to-extracted-file/self-autoload.php");

      use LeviZwannah\MpesaSdk\Mpesa;

      $mpesa = Mpesa::new();
      ...
    ?>

Getting Started

To use the SDK you must first ensure that all the classes are included. This can be done with an autoloader. If you are using composer, then the default vendor/autoload.php file will do the job. However, if you downloaded the SDK manually, then the self-autoload.php file will do the job. All you need to do is to include the autoloader files. 

<?php
    require_once(__DIR__ . "/vendor/autoload.php") // if using composer
    //or
    require_once(__DIR__ . "/extracted-folder/self-autoload.php") // if you manually downloaded the sdk
?>

The namespace for the SDK is LeviZwannah\MpesaSdk. The main class is the Mpesa class. Every API provided by Safaricom Mpesa is wrapped in a class that extends the Mpesa class. For example, the STK class is defined like this: class STK extends Mpesa {...}. However, to access the STK object, use $mpesa->stk() function.

Warning

Do not create an object of the children classes directly. Always use the $mpesa object. For example, to get an $stk object, use $mpesa->stk();

Info

All the subclasses extend the Mpesa class, so they have all their parent's properties and functions available to them. For example, you can get a $reversal object from a $stk object using the $stk->reversal(). However, this is not recommended because some configurations will not be provided to the returned $reversal object. Therefore, always use the $mpesa object to get the other objects.

See below

Naming

In the SDK, every setter method has the same name as the value it is setting. For example, the setter for till number is till(string $tillNumber). Therefore, to set the value of the till number you use $mpesa->till("xxxxxxx");.

To keep the method names short, properties like short code are changed to code. Therefore, to set the business short code, use the $mpesa->code("xxxxxx") method. The consumer key is shortened to key and the consumer secret is shortened to secret.

$mpesa->configure($config)

To provide flexibility and ease of use, the configure method will set the properties of the instance on which it is called. For example, $mpesa->configure(["key" => "consumer key", "secret" => "consumer secret"]) will set the $mpesa->key and $mpesa->secret properties respectively. In short, the keys of the array passed to the configure method represents properties of the $mpesa object that will be set to the corresponding values. This is also true for any subclass of the Mpesa class such as the STK, Reversal, etc classes.

Note

Also remember that you can use properties to do exactly what the configure method is doing. For example, this will achieve the same result as the above: $mpesa->key("Consumer Key")->secret("Consumer Secret");. You choose!

Putting it all together

Common methods

For flexibility and openness, there are methods that can be used as standalone in the SDK. These methods are from the Mpesa class therefore is available to every subclass. For example, you can develop a custom class that extends the SDK's Mpesa class or maybe use some individual functionalities to achieve something different. this section covers the common methods.

Note

Most of the common methods require that the consumer key, consumer secret, and environment of the $mpesa object be set before they can work correctly. You can use the $mpesa->configure(array $config) method to set them or rather use the properties.

$mpesa->token()

This method returns the Mpesa access token based on the $mpesa->env value. If the environment is sandbox, then you get a sandbox access token, otherwise, you get a live token.

$mpesa->request(array $data, string $path)

This method abstracts the making of request to the Safaricom Mpesa server. It firstly request an access token before making the request, sending along with it the access token. The first argument of the method is an associative array containing key-value pairs of the data to send. The second argument is a string containing the REST path to the service you want to use. For example, to make an Stk push request, the path will be "/mpesa/stkpush/v1/processrequest". This method returns the JSON decoded response from Safaricom Mpesa server. Hence, it returns an Object.

$mpesa->error()

This method returns RequestError or false if the $mpesa->response object contains an errorCode field -- checking if the request to Mpesa was successful. Please note that if you use the $mpesa->request() function, you must manually set the response of the $mpesa object like so $mpesa->response($response). A RequestError object contains two field, $error->code: The error code and $error->message: The error message.

$mpesa->accepted()

This method checks if the mpesa request was accepted for processing. When a request is accepted for processing, the response code of the response is 0. Hence, this method checks if the response code is 0. Again, this is similar to $mpesa->error() and requires that a response object be set.

Mpesa::data($asArray = false)

This function will return the json decoded data from the Safaricom Mpesa request in your callback handler. The returned data can be an object or an associative array based on the $asArray parameter.

STK Push

This section teaches you how to make STK push using the SDK. This will be mostly through code and comments in the code. All the properties of an Stk object can be set using the $stk->configure($config) method. Also, all the properties can be chained when used as setters. All STK pushes require a phone number, an amount, and a callback URL. The callback URL must be a full url.

Note

Every STK push requires a transaction ID: CustomerBuyGoodsOnline or CustomerPaybillOnline. The difference between the two is that, if the type is CustomerBuyGoodsOnline, then you need to have a till number set before calling the $stk->push() method. To set the transaction ID, call $stk->buygoods() or $stk->paybill().

STK push for Till Numbers

When making an STK push request for a till number, you are not required to set the reference (the account number). The transaction description is always optional. However, you must set the till number as well ensure that $stk->buygoods() is called before calling the push method.

Let the code speak for itself

STK push for Paybill Numbers

In the real world, paybill payments require an account number. You can set that here using $stk->reference("account number") -- defaults to "payment". The transaction description is always optional. Paybill STK pushes do not require a till number, but you must ensure that you call $stk->paybill() before calling the push method.

Let the code speak for itself

Chaining vs Configure

You can always use the configure method. This has been said 1M times throughout this documentation. All you need is to ensure that the keys in the configuration array match the properties you are trying to set. Also, you can use method chaining to do all the work in one line of code.
